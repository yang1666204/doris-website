AI技術が前例のないペースで進歩し続ける中、データインフラストラクチャは現代のAIアプリケーションの基盤となっています。高性能リアルタイム分析データベースであるApache Dorisは、全文検索、ベクトル検索、AI機能、MCPベースのインテリジェントインタラクションのネイティブ統合を提供しています。これらの機能は連携して、ストレージ、検索、分析を包含する包括的なAIデータスタックを形成します。

- [full-text search](text-search/overview.md)
- [vector search](vector-search/overview.md)
- [AI functions](ai-function-overview.md)
- [Doris MCP サーバー](https://github.com/apache/doris-mcp-server)

Dorisは、ハイブリッド検索・分析、エージェント向けデータ分析、セマンティック検索、RAGアプリケーション開発、大規模AIシステムのオブザーバビリティなど、幅広いAI駆動ワークロード向けに統合された高性能でコスト効率の良いソリューションを提供します。

## エージェント向け分析

AI エージェント技術の台頭により、ますます多くの分析的意思決定がAIによって自動的に完了されるようになり、データプラットフォームには究極のリアルタイム性能と高同時実行性能が求められています。従来の「手動分析」とは異なり、エージェント向け分析では、データクエリと意思決定をミリ秒スケールで完了し、大量のエージェントからの同時アクセスをサポートする必要があります。典型的なシナリオには、リアルタイム不正検知、インテリジェント広告配信、パーソナライズドレコメンデーションなどがあります。

Dorisは、高性能MPPアーキテクチャにより、これらのエージェント向け分析シナリオで優れた優位性を実証しています：

- **リアルタイム取り込み・更新**：エージェントの意思決定が最新のデータに基づくことを保証、最小データレイテンシ約1秒

- **超高速分析**：平均クエリレイテンシ < 100ms、エージェントのリアルタイム意思決定要件を満たす
- **高同時実行クエリ**：10,000+ QPSをサポート、大量のエージェント同時クエリを容易に処理
- **ネイティブエージェント統合**：MCP サーバーを通じてAI エージェントとシームレスに統合、開発・統合ワークフローを簡素化

## ハイブリッド検索・分析処理

![img](/images/vector-search/image-5.png)

半構造化データおよび非構造化データは、データ分析において第一級市民になりつつあります。顧客レビュー、チャットログ、プロダクションログ、車両信号などのデータは、ビジネス意思決定プロセスに深く統合されています。従来の構造化分析ソリューションは、全文検索とベクトル検索機能を組み込み、セマンティック検索をサポートしながら、同一プラットフォーム上で多次元分析と集約統計を可能にする必要があります。例としては：

- **顧客インサイト**：レビューテキスト検索とユーザー行動分析を組み合わせて、顧客ニーズと満足度トレンドを正確に特定
- **スマートマニュファクチャリング**：プロダクションログ全文検索、設備画像認識、IoTメトリクス分析を統合して、故障予測と品質最適化を実現
- **車両インターネット**：車両信号データ分析、ユーザーフィードバックテキストマイニング、運転行動ベクトル検索を統合して、スマートコックピット体験を向上

Dorisの高性能リアルタイム分析、テキストインデクシング、ベクトルインデクシング機能に基づいて上記シナリオ向けのAIアプリケーションを構築することで、複数の利点が得られます：

- **統合アーキテクチャ**：単一プラットフォームで構造化分析、全文検索、ベクトル検索を処理、データ移行と異種システム統合を排除
- **ハイブリッドクエリ性能**：単一SQLでベクトル類似性検索、キーワードフィルタリング、集約分析を同時実行し、優れたクエリ性能を実現
- **柔軟なスキーマサポート**：VARIANT型が動的JSON構造をネイティブサポート、Light Schema Changeが秒レベルのフィールドとインデックス変更を実現
- **フルスタック最適化**：転置インデックスとベクトルインデックスからMPP実行エンジンまでのエンドツーエンド最適化により、検索精度と分析効率のバランスを実現

## AI向けレイクハウス

AIモデルとアプリケーション開発では、大規模データセットから訓練セットの準備、特徴エンジニアリング、データ品質評価を行う必要があります。従来のアーキテクチャでは、データレイクと分析エンジン間で頻繁なデータ移行が必要になることがよくあります。レイクハウスアーキテクチャは、データレイクのオープンストレージとリアルタイム分析エンジンを深く統合し、データ準備、特徴エンジニアリング、モデル評価の全ワークフローを統合プラットフォーム上でサポートし、データサイロを排除してAI開発反復を加速します。

- **レイクハウス統合アーキテクチャ**：オープンテーブル形式（IcebergやPaimonなど）とカタログに基づいてオープンlakehouseを構築し、分析データとAIデータを統一管理
- **リアルタイム分析エンジン**：Dorisがリアルタイム分析エンジンとして機能し、対話型クエリと軽量ETLをサポート、データ準備と特徴エンジニアリングに最高速のSQL計算能力を提供
- **シームレスなデータフロー**：データ移動なしでデータレイクの直接読み書き、ストレージ層での統一管理と計算層での柔軟な高速化

Dorisベースのレイクハウスアーキテクチャは、AI全体のワークフローを加速します：

- **大規模データ準備**：Dorisの効率的なデータ処理能力を活用して、PBスケールのデータレイクからデータをフィルタリング、サンプリング、クレンジングし、高品質な訓練データセットを迅速に構築
- **リアルタイム特徴エンジニアリング**：Dorisのリアルタイム分析能力を活用してオンライン特徴抽出、変換、集約計算を実行し、モデル訓練と推論にリアルタイム特徴サービスを提供
- **品質評価**：テストセットとプロダクションデータに対して多次元の高速分析を実施し、モデル性能とデータドリフトを継続的に監視

## RAG（Retrieval-Augmented Generation）

RAGは外部ナレッジベースから関連情報を検索して大規模モデルにコンテキストを提供し、モデルのハルシネーションとナレッジ通貨問題を効果的に解決します。ベクトルエンジンはRAGシステムのコアコンポーネントであり、大規模ナレッジベースから最も関連性の高い文書フラグメントを迅速に呼び出し、高同時実行ユーザークエリリクエストをサポートしてアプリケーションの応答性を確保する必要があります。

- **企業ナレッジ**：内部文書とマニュアルに基づいたインテリジェントQ&Aシステムの構築により、従業員が自然言語を通じて正確な回答を迅速に取得可能
- **インテリジェント顧客サービスアシスタント**：製品ナレッジベースと履歴ケースを組み合わせて、カスタマーサービス担当者またはチャットボットに正確な応答提案を提供
- **インテリジェント文書アシスタント**：大規模文書コレクション内の関連コンテンツを迅速に特定し、研究、執筆、意思決定プロセスを支援

Dorisベースでこれらのシナリオ向けにRAGアプリケーションを構築することで、以下の利点が得られます：

- **高同時実行性能**：分散アーキテクチャが高同時実行ベクトル検索をサポート、大規模同時実行ユーザーアクセスを容易に処理
- **ハイブリッド検索能力**：単一SQLでベクトル類似性検索とキーワードフィルタリングを同時実行し、セマンティック呼び出しと完全一致のバランスを実現
- **弾性スケーリング**：クラスタ拡張に伴いクエリ性能が線形スケール、数百万から数百億ベクトルへのシームレスな移行
- **統合ソリューション**：ベクトルデータ、オリジナル文書、ビジネスデータを統一管理し、RAGアプリケーションのデータアーキテクチャを簡素化

## AIオブザーバビリティ

AIモデル訓練反復とアプリケーション運用は、大量のログ、メトリクス、トレーシングデータを生成します。問題を正確に特定し性能を継続的に最適化するために、オブザーバビリティシステムはAIインフラストラクチャの重要なコンポーネントとなっています。ビジネス規模の拡大に伴い、オブザーバビリティプラットフォームは、PBスケールデータの高スループット書き込み、ミリ秒レベル検索応答、コスト制御など、複数の課題に直面しています。典型的なユースケースには：

- **モデル訓練監視**：訓練メトリクスとリソース消費のリアルタイム追跡、訓練異常と性能ボトルネックの迅速な特定
- **推論サービストレーシング**：各推論リクエストの完全なトレースを記録し、レイテンシソースとエラーパターンを分析
- **AI****アプリケーションログ分析**：大量のアプリケーションログの全文検索と集約分析により、トラブルシューティングと行動インサイトをサポート

DorisでAIオブザーバビリティを構築することで、以下の利点が得られます：

- **究極の性能**：PB/day（10GB/s）の持続書き込みをサポート、転置インデックスがログ検索を加速して秒レベル応答を実現
- **コスト最適化**：5:1から10:1の圧縮率、ストレージコスト50%-80%削減、コールドデータの低コストストレージをサポート
- **柔軟なスキーマ**：Light Schema Changeが秒レベルフィールド変更を実現、VARIANT型が動的JSON構造をネイティブサポート
- **エコシステムフレンドリー**：OpenTelemetryとELKエコシステムに対応、Grafana/Kibana可視化ツールとの統合をサポート

## セマンティック検索

セマンティック検索は、ベクトル化技術を通じてテキストの深い意味を捉えます。クエリ用語が文書の文言と異なる場合でも、意味的に関連するコンテンツを検索できます。これは、言語横断検索、同義語認識、意図理解などのシナリオにとって重要であり、検索再現率とユーザーエクスペリエンスを大幅に改善します。典型的なユースケースには：

- **企業文書検索**：従業員が自然言語で問題を記述し、システムが意図を理解して大量の文書から意味的に関連するポリシー、手順、ナレッジを呼び出し
- **Eコマース製品検索**：ユーザーが「夏に適した通気性のある靴」を入力し、システムがニーズを理解して単なるキーワードマッチングではなく関連製品を呼び出し
- **コンテンツレコメンデーション**：記事や動画のセマンティック類似性に基づくインテリジェントレコメンデーション、異なる文言でも潜在的に興味のあるコンテンツを発見

Dorisベースでセマンティック検索アプリケーションを構築することで、以下の利点が得られます：

- **高性能ベクトル検索**：HNSWとIVFアルゴリズムをサポート、億スケールベクトルに対して1秒未満の応答、大規模セマンティック検索要件を容易に処理
- **強化されたハイブリッド検索**：単一SQLでセマンティック検索とキーワードフィルタリングを統合し、必要な語彙ヒットを確保しながらセマンティックに関連するコンテンツを呼び出し
- **マルチモーダル拡張**：テキストセマンティック検索だけでなく、画像や音声などのマルチモーダルコンテンツのセマンティック検索にも拡張可能
- **柔軟な量子化最適化**：SQ/PQ量子化技術により、検索精度を維持しながらストレージと計算コストを大幅に削減
