AI技術が前例のない速度で進歩し続ける中、データインフラストラクチャは現代のAIアプリケーションの基盤となっています。Apache Dorisは高性能リアルタイム分析データベースとして、全文検索、ベクトル検索、AI機能、MCPベースのインテリジェント連携のネイティブ統合を提供します。これらの機能を組み合わせることで、ストレージ、検索、分析を包含する包括的なAIデータスタックを形成します。

- [full-text search](text-search/overview.md)
- [vector search](vector-search/overview.md)
- [AI functions](ai-function-overview.md)
- [Doris MCP サーバー](https://github.com/apache/doris-mcp-server)

Dorisは、ハイブリッド検索・分析、エージェント向けデータ分析、セマンティック検索、RAGアプリケーション開発、大規模AIシステムの可観測性など、幅広いAI駆動ワークロードに対して、統一された高性能でコスト効率の高いソリューションを提供します。

## エージェント Facing Analytics

AI エージェント技術の興隆に伴い、ますます多くの分析的意思決定がAIによって自動的に完了されるようになり、データプラットフォームには究極のリアルタイム性能と高同時実行性能の提供が求められます。従来の「手動分析」とは異なり、エージェント Facing Analyticsでは、データクエリと意思決定がミリ秒スケールで完了し、大量のエージェントからの同時アクセスをサポートすることが必要です。典型的なシナリオには、リアルタイム詐欺検出、インテリジェント広告配信、パーソナライズドレコメンデーションなどがあります。

Dorisは、高性能MPPアーキテクチャにより、これらのエージェント向け分析シナリオにおいて優れた利点を示します：

- **リアルタイム取り込み・更新**：エージェントの意思決定が最新データに基づくことを保証、約1秒の最小データレイテンシ

- **超高速分析**：平均クエリレイテンシ100ms未満、エージェントのリアルタイム意思決定要件を満たす
- **高同時実行クエリ**：10,000+ QPSをサポート、大量のエージェント同時クエリを容易に処理
- **ネイティブエージェント統合**：MCP サーバーを通じてAI エージェントとシームレスに統合、開発・統合ワークフローを簡素化

## Hybrid Search and Analytics Processing

![img](/images/vector-search/image-5.png)

半構造化データと非構造化データは、データ分析における第一級市民となりつつあります。顧客レビュー、チャットログ、プロダクションログ、車両信号などのデータは、ビジネス意思決定プロセスに深く統合されています。従来の構造化分析ソリューションは、全文検索およびベクトル検索機能を組み込み、セマンティック検索をサポートしながら、同一プラットフォーム上で多次元分析と集計統計を可能にする必要があります。例として：

- **顧客インサイト**：レビューテキスト検索とユーザー行動分析を組み合わせて、顧客ニーズと満足度トレンドを正確に特定
- **スマートマニュファクチャリング**：プロダクションログ全文検索、設備画像認識、IoTメトリック分析を統合して、故障予測と品質最適化を実現
- **Internet of Vehicles**：車両信号データ分析、ユーザーフィードバックテキストマイニング、運転行動ベクトル検索を総合して、スマートコックピット体験を向上

Dorisの高性能リアルタイム分析、テキストインデックス、ベクトルインデックス機能に基づいて上記シナリオのAIアプリケーションを構築することは、複数の利点を提供します：

- **統一アーキテクチャ**：構造化分析、全文検索、ベクトル検索を単一プラットフォームで処理、データ移行と異種システム統合を排除
- **ハイブリッドクエリ性能**：単一SQLでベクトル類似検索、キーワードフィルタリング、集計分析を同時実行、優れたクエリ性能
- **柔軟なスキーマサポート**：VARIANT型が動的JSON構造をネイティブサポート、Light Schema Changeが秒レベルのフィールドとインデックス変更を実現
- **フルスタック最適化**：転置インデックスとベクトルインデックスからMPP実行エンジンまでのエンドツーエンド最適化、検索精度と分析効率のバランス

## レイクハウス for AI

AIモデル・アプリケーション開発では、大量データセットからトレーニングセットの準備、特徴量エンジニアリング、データ品質評価を実行する必要があります。従来のアーキテクチャでは、データレイクと分析エンジン間での頻繁なデータ移行が必要でした。レイクハウスアーキテクチャは、データレイクのオープンストレージとリアルタイム分析エンジンを深く統合し、データ準備、特徴量エンジニアリング、モデル評価の全ワークフローを統一プラットフォーム上でサポート、データサイロを排除してAI開発イテレーションを加速します。

- **レイクハウス統一アーキテクチャ**：オープンテーブルフォーマット（Iceberg/Paimonなど）とカタログに基づくオープンlakehouseを構築、分析データとAIデータを統一管理
- **リアルタイム分析エンジン**：Dorisがリアルタイム分析エンジンとして機能、インタラクティブクエリと軽量ETLをサポート、データ準備と特徴量エンジニアリングに最高速のSQL計算能力を提供
- **シームレスデータフロー**：データ移動なしにデータレイクへの直接読み書き、ストレージ層での統一管理と計算層での柔軟な加速

Dorisに基づくレイクハウスアーキテクチャは、AI全体ワークフローを加速します：

- **大規模データ準備**：Dorisのエフィシェントなデータ処理能力を活用して、PBスケールデータレイクからデータのフィルタリング、サンプリング、クレンジングを実行、高品質トレーニングデータセットを迅速構築
- **リアルタイム特徴量エンジニアリング**：Dorisのリアルタイム分析能力を利用してオンライン特徴量抽出、変換、集計計算を実行、モデルトレーニングと推論にリアルタイム特徴量サービスを提供
- **品質評価**：テストセットとプロダクションデータに対する多次元迅速分析を実行、モデル性能とデータドリフトを継続監視

## RAG (Retrieval-Augmented Generation)

RAGは外部ナレッジベースから関連情報を検索して大規模モデルにコンテキストを提供し、モデルの幻覚と知識通貨問題を効果的に解決します。ベクトルエンジンはRAGシステムのコアコンポーネントであり、大量のナレッジベースから最も関連性の高い文書フラグメントの迅速な再現と、アプリケーション応答性確保のための高同時実行ユーザークエリリクエストのサポートが必要です。

- **企業ナレッジ**：内部文書とマニュアルに基づくインテリジェントQ&Aシステム構築、従業員が自然言語により迅速に正確な回答を取得可能
- **インテリジェントカスタマーサービスアシスタント**：製品ナレッジベースと履歴ケースを組み合わせて、カスタマーサービス担当者やチャットボットに正確な応答提案を提供
- **インテリジェント文書アシスタント**：大規模文書コレクション内の関連コンテンツを迅速に特定、研究、執筆、意思決定プロセスを支援

これらのシナリオでDorisに基づくRAGアプリケーション構築は、以下の利点を提供します：

- **高同時実行性能**：分散アーキテクチャが高同時実行ベクトル検索をサポート、大規模同時ユーザーアクセスを容易に処理
- **ハイブリッド検索機能**：単一SQLでベクトル類似検索とキーワードフィルタリングを同時実行、セマンティック再現と完全マッチングのバランス
- **エラスティックスケーリング**：クエリ性能がクラスター拡張と線形スケール、数百万から数百億ベクトルへのシームレス移行
- **統一ソリューション**：ベクトルデータ、原文書、ビジネスデータを統一管理、RAGアプリケーションのデータアーキテクチャを簡素化

## AI Observability

AIモデルトレーニングイテレーションとアプリケーション運用は、大量のログ、メトリック、トレーシングデータを生成します。問題を正確に特定し性能を継続的に最適化するために、可観測性システムはAIインフラストラクチャの重要コンポーネントとなっています。ビジネス規模拡大に伴い、可観測性プラットフォームはPBスケールデータの高スループット書き込み、ミリ秒レベル検索応答、コスト制御などの複数の課題に直面します。典型的な使用例：

- **モデルトレーニング監視**：トレーニングメトリックとリソース消費のリアルタイム追跡、トレーニング異常と性能ボトルネックの迅速特定
- **推論サービストレーシング**：各推論リクエストの完全トレース記録、レイテンシソースとエラーパターンの分析
- **AI****アプリケーションログ分析**：大量アプリケーションログの全文検索と集計分析、トラブルシューティングと行動インサイトをサポート

DorisによるAI Observability構築は、以下の利点を提供します：

- **究極の性能**：PB/day（10GB/s）の持続書き込みをサポート、転置インデックスがログ検索を加速して秒レベル応答
- **コスト最適化**：5:1から10:1の圧縮率、ストレージコスト50%-80%削減、コールドデータの低コストストレージをサポート
- **柔軟なスキーマ**：Light Schema Changeが秒レベルフィールド変更を実現、VARIANT型が動的JSON構造をネイティブサポート
- **エコシステムフレンドリー**：OpenTelemetryとELKエコシステムと互換、Grafana/Kibana可視化ツールとの統合をサポート

## Semantic Search

セマンティック検索は、ベクトル化技術を通じてテキストの深い意味を捉えます。クエリ用語が文書の文言と異なる場合でも、セマンティカルに関連するコンテンツを依然として検索できます。これは、クロス言語検索、同義語認識、意図理解などのシナリオにとって重要であり、検索再現率とユーザー体験を大幅に向上させます。典型的な使用例：

- **企業文書検索**：従業員が自然言語で問題を記述し、システムが意図を理解して大量文書からセマンティカルに関連するポリシー、手順、ナレッジを再現
- **Eコマース商品検索**：ユーザーが「夏に適した通気性の良い靴」と入力し、システムがニーズを理解して単なるキーワードマッチングではなく関連商品を再現
- **コンテンツレコメンデーション**：記事と動画のセマンティック類似性に基づくインテリジェントレコメンデーション、異なる文言での潜在的興味コンテンツを発見

Dorisに基づくセマンティック検索アプリケーション構築は、以下の利点を提供します：

- **高性能ベクトル検索**：HNSWとIVFアルゴリズムをサポート、億スケールベクトルで秒未満応答、大規模セマンティック検索要件を容易に処理
- **強化ハイブリッド検索**：単一SQLでセマンティック検索とキーワードフィルタリングを統合、必要な語彙ヒットを確保しながらセマンティカルに関連するコンテンツを再現
- **マルチモーダル拡張**：テキストセマンティック検索だけでなく、画像、音声などのマルチモーダルコンテンツのセマンティック検索にも拡張可能
- **柔軟な量子化最適化**：SQ/PQ量子化技術により、検索精度を維持しながらストレージと計算コストを大幅削減
