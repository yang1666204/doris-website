AI技術が前例のないペースで進歩し続ける中、データインフラストラクチャは現代のAIアプリケーションの基盤となっています。高性能リアルタイム分析データベースであるApache Dorisは、全文検索、ベクトル検索、AI機能、MCPベースのインテリジェントな対話のネイティブ統合を提供しています。これらの機能は連携して、ストレージ、検索、分析にまたがる包括的なAIデータスタックを形成します。

- [full-text search](text-search/overview.md)
- [vector search](vector-search/overview.md)
- [AI functions](ai-function-overview.md)
- [Doris MCP サーバー](https://github.com/apache/doris-mcp-server)

Dorisは、ハイブリッド検索と分析、エージェント対応データ分析、セマンティック検索、RAGアプリケーション開発、大規模AIシステムのオブザーバビリティなど、幅広いAI駆動ワークロードに対して統一された高性能かつコスト効率的なソリューションを提供します。

## エージェント Facing Analytics

AI エージェント技術の台頭により、ますます多くの分析的意思決定がAIによって自動的に完了されるようになり、データプラットフォームには究極のリアルタイムパフォーマンスと高い同時実行機能の提供が求められています。従来の「手動分析」とは異なり、エージェント Facing Analyticsではデータクエリと意思決定をミリ秒スケールで完了し、大量のエージェントからの同時アクセスをサポートする必要があります。典型的なシナリオには、リアルタイム不正検出、インテリジェント広告配置、パーソナライズされた推奨などがあります。

Dorisは高性能MPPアーキテクチャにより、これらのエージェント対応分析シナリオで優れた優位性を示しています：

- **リアルタイム取り込み・更新**：エージェントの意思決定が最新データに基づくことを保証、最小データ遅延は約1秒

- **超高速分析**：平均クエリレイテンシー < 100ms、エージェントのリアルタイム意思決定要件を満たす
- **高同時実行クエリ**：10,000+ QPSをサポート、大量のエージェント同時クエリを容易に処理
- **ネイティブエージェント統合**：MCP サーバーを通じてAI エージェントとシームレスに統合、開発と統合ワークフローを簡素化

## Hybrid Search and Analytics Processing

![img](/images/vector-search/image-5.png)

半構造化データと非構造化データは、データ分析における第一級市民となっています。顧客レビュー、チャットログ、プロダクションログ、車両信号、その他のデータがビジネス意思決定プロセスに深く統合されています。従来の構造化分析ソリューションは、全文検索とベクトル検索機能を組み込み、セマンティック検索をサポートしながら、同一プラットフォーム上で多次元分析と集計統計を可能にする必要があります。例えば：

- **顧客インサイト**：レビューテキスト検索とユーザー行動分析を組み合わせて、顧客ニーズと満足度トレンドを正確に特定
- **スマート製造**：プロダクションログ全文検索、設備画像認識、IoTメトリクス分析を統合し、故障予測と品質最適化を実現
- **Internet of Vehicles**：車両信号データ分析、ユーザーフィードバックテキストマイニング、運転行動ベクトル検索を統合し、スマートコックピット体験を向上

Dorisの高性能リアルタイム分析、テキストインデックス、ベクトルインデックス機能に基づいて上記シナリオのAIアプリケーションを構築することで、複数の利点があります：

- **統一アーキテクチャ**：構造化分析、全文検索、ベクトル検索を単一プラットフォームで処理、データ移行と異種システム統合を排除
- **ハイブリッドクエリパフォーマンス**：単一SQLでベクトル類似度検索、キーワードフィルタリング、集計分析を同時実行し、優秀なクエリパフォーマンスを実現
- **柔軟なスキーマサポート**：VARIANTタイプが動的JSON構造をネイティブサポート、Light Schema Changeが秒レベルでのフィールドとインデックス変更を可能に
- **フルスタック最適化**：転置インデックスとベクトルインデックスからMPP実行エンジンまでのエンドツーエンド最適化、検索精度と分析効率のバランスを実現

## レイクハウス for AI

AIモデルとアプリケーション開発には、大規模データセットからの訓練セット準備、特徴エンジニアリング、データ品質評価が必要です。従来のアーキテクチャでは、データレイクと分析エンジン間で頻繁なデータ移行が必要でした。レイクハウスアーキテクチャは、データレイクのオープンストレージとリアルタイム分析エンジンを深く統合し、統一プラットフォーム上でデータ準備、特徴エンジニアリング、モデル評価の全ワークフローをサポートし、データサイロを排除してAI開発イテレーションを加速します。

- **レイクハウス統一アーキテクチャ**：オープンテーブル形式（IcebergやPaimonなど）とカタログに基づいてオープンlakehouseを構築、分析データとAIデータを統一管理
- **リアルタイム分析エンジン**：Dorisがリアルタイム分析エンジンとして機能し、対話型クエリと軽量ETLをサポート、データ準備と特徴エンジニアリングに最速のSQLコンピューティング機能を提供
- **シームレスなデータフロー**：データ移動なしでデータレイクの直接読み書き、ストレージ層での統一管理とコンピュート層での柔軟な加速

Dorisに基づくレイクハウスアーキテクチャはAIワークフロー全体を加速します：

- **大規模データ準備**：Dorisの効率的なデータ処理機能を活用してPBスケールのデータレイクからデータをフィルタリング、サンプリング、クレンジングし、高品質な訓練データセットを迅速に構築
- **リアルタイム特徴エンジニアリング**：Dorisのリアルタイム分析機能を活用してオンライン特徴抽出、変換、集計コンピューティングを実行し、モデル訓練と推論にリアルタイム特徴サービスを提供
- **品質評価**：テストセットとプロダクションデータで多次元迅速分析を実行、モデルパフォーマンスとデータドリフトを継続的に監視

## RAG (Retrieval-Augmented Generation)

RAGは外部知識ベースから関連情報を取得して大規模モデルにコンテキストを提供し、モデルの幻覚と知識の時効性の問題を効果的に解決します。ベクトルエンジンはRAGシステムの中核コンポーネントであり、大規模知識ベースから最も関連性の高いドキュメント断片を迅速にリコールし、高同時実行ユーザークエリリクエストをサポートしてアプリケーションの応答性を確保する必要があります。

- **企業知識**：社内ドキュメントとマニュアルに基づいてインテリジェントQ&Aシステムを構築、従業員が自然言語で迅速に正確な回答を取得
- **インテリジェント顧客サービスアシスタント**：製品知識ベースと過去の事例を組み合わせて、顧客サービス担当者やチャットボットに正確な回答提案を提供
- **インテリジェントドキュメントアシスタント**：大規模ドキュメントコレクション内の関連コンテンツを迅速に特定し、研究、執筆、意思決定プロセスを支援

Dorisに基づいてRAGアプリケーションを構築することで、これらのシナリオで以下の利点があります：

- **高同時実行パフォーマンス**：分散アーキテクチャが高同時実行ベクトル検索をサポート、大規模同時ユーザーアクセスを容易に処理
- **ハイブリッド検索機能**：単一SQLでベクトル類似度検索とキーワードフィルタリングを同時実行、セマンティックリコールと完全一致のバランスを実現
- **弾性スケーリング**：クエリパフォーマンスがクラスター拡張とともに線形スケール、数百万から数百億ベクトルへシームレスに移行
- **統一ソリューション**：ベクトルデータ、元ドキュメント、ビジネスデータを統一管理、RAGアプリケーションのデータアーキテクチャを簡素化

## AI Observability

AIモデル訓練イテレーションとアプリケーション運用は大量のログ、メトリクス、トレーシングデータを生成します。問題を正確に特定し、パフォーマンスを継続的に最適化するため、オブザーバビリティシステムはAIインフラストラクチャの重要コンポーネントとなっています。ビジネス規模の拡大に伴い、オブザーバビリティプラットフォームはPBスケールデータの高スループット書き込み、ミリ秒レベル検索応答、コスト制御など複数の課題に直面しています。典型的なユースケースには以下があります：

- **モデル訓練監視**：訓練メトリクスとリソース消費のリアルタイム追跡、訓練異常とパフォーマンスボトルネックの迅速な特定
- **推論サービストレーシング**：各推論リクエストの完全なトレースを記録、レイテンシーソースとエラーパターンの分析
- **AI** **アプリケーションログ分析**：大量アプリケーションログの全文検索と集計分析、トラブルシューティングと行動インサイトをサポート

DorisでAI Observabilityを構築することで以下の利点があります：

- **究極のパフォーマンス**：PB/day（10GB/s）の持続的書き込みをサポート、転置インデックスがログ検索を加速し秒レベル応答を実現
- **コスト最適化**：5:1から10:1の圧縮比、50%-80%のストレージコスト削減、コールドデータの低コストストレージをサポート
- **柔軟なスキーマ**：Light Schema Changeが秒レベルでのフィールド変更を可能にし、VARIANTタイプが動的JSON構造をネイティブサポート
- **エコシステムフレンドリー**：OpenTelemetryとELKエコシステムと互換性があり、Grafana/Kibana可視化ツールとの統合をサポート

## Semantic Search

セマンティック検索はベクトル化技術を通じてテキストの深い意味を捉えます。クエリ用語がドキュメントの文言と異なっていても、セマンティックに関連するコンテンツを検索できます。これは言語間検索、同義語認識、意図理解などのシナリオにおいて重要で、検索リコール率とユーザー体験を大幅に改善します。典型的なユースケースには以下があります：

- **企業ドキュメント検索**：従業員が自然言語で問題を説明すると、システムが意図を理解して大量ドキュメントからセマンティックに関連するポリシー、手順、知識をリコール
- **Eコマース製品検索**：ユーザーが「夏に適した通気性の良い靴」と入力すると、システムがニーズを理解して単なるキーワードマッチングではなく関連製品をリコール
- **コンテンツ推奨**：記事や動画のセマンティック類似性に基づくインテリジェント推奨、異なる表現でも潜在的に興味のあるコンテンツを発見

Dorisに基づいてセマンティック検索アプリケーションを構築することで以下の利点があります：

- **高性能ベクトル検索**：HNSWとIVFアルゴリズムをサポート、数億スケールベクトルに対して秒未満応答、大規模セマンティック検索要件を容易に処理
- **強化されたハイブリッド検索**：単一SQLでセマンティック検索とキーワードフィルタリングを統合、必要な語彙ヒットを確保しながらセマンティックに関連するコンテンツをリコール
- **マルチモーダル拡張**：テキストセマンティック検索のみならず、画像や音声などマルチモーダルコンテンツのセマンティック検索にも拡張可能
- **柔軟な量子化最適化**：SQ/PQ量子化技術により、検索精度を維持しながらストレージと計算コストを大幅削減
