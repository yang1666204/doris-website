AI技術が未曾有のペースで進歩し続ける中、データインフラストラクチャは現代のAIアプリケーションの基盤となっています。Apache Dorisは高性能リアルタイム分析データベースとして、全文検索、ベクトル検索、AI関数、MCPベースのインテリジェントインタラクションのネイティブ統合を提供します。これらの機能により、ストレージ、検索、分析にわたる包括的なAIデータスタックを形成しています。

- [全文検索](text-search/overview.md)
- [ベクトル検索](vector-search/overview.md)
- [AI関数](ai-function-overview.md)
- [Doris MCP サーバー](https://github.com/apache/doris-mcp-server)

Dorisは、ハイブリッド検索・分析、エージェント向けデータ分析、セマンティック検索、RAGアプリケーション開発、大規模AIシステムの可観測性など、幅広いAI駆動ワークロードに対して統合的で高性能かつコスト効率的なソリューションを提供します。

## エージェント Facing Analytics

AI エージェント技術の台頭により、ますます多くの分析的意思決定がAIによって自動的に完了されるようになり、データプラットフォームには究極のリアルタイム性能と高い並行性能力が求められています。従来の「手動分析」とは異なり、エージェント Facing Analyticsではデータクエリと意思決定をミリ秒スケールで完了し、大量のエージェントからの並行アクセスをサポートする必要があります。典型的なシナリオには、リアルタイム不正検知、インテリジェント広告配信、パーソナライズド推薦などがあります。

Dorisは高性能MPPアーキテクチャにより、これらのエージェント向け分析シナリオで優れた優位性を実証しています：

- **リアルタイム取り込み・更新**：エージェントの決定が最新のデータに基づくことを保証、最小データレイテンシ約1秒

- **超高速分析**：平均クエリレイテンシ < 100ms、エージェントのリアルタイム決定要件を満たす
- **高並行クエリ**：10,000+ QPSをサポート、大量エージェント並行クエリを容易に処理
- **ネイティブエージェント統合**：MCP サーバーを通じてAI エージェントとシームレスに統合、開発・統合ワークフローを簡素化

## ハイブリッド検索・分析処理

![img](/images/vector-search/image-5.png)

半構造化・非構造化データは、データ分析におけるファーストクラスシチズンとなっています。顧客レビュー、チャットログ、本番ログ、車両信号などのデータは、ビジネス意思決定プロセスに深く統合されています。従来の構造化分析ソリューションには、全文検索とベクトル検索機能を組み込み、セマンティック検索をサポートしつつ、同一プラットフォームで多次元分析と集約統計を可能にすることが必要です。例：

- **顧客インサイト**：レビューテキスト検索とユーザー行動分析を組み合わせ、顧客ニーズと満足度トレンドを正確に特定
- **スマート製造**：本番ログ全文検索、設備画像認識、IoTメトリクス分析を統合し、故障予測と品質最適化を実現
- **コネクテッドカー**：車両信号データ分析、ユーザーフィードバックテキストマイニング、運転行動ベクトル検索を統合し、スマートコックピット体験を向上

上記シナリオでDorisの高性能リアルタイム分析、テキストインデックス、ベクトルインデックス機能に基づくAIアプリケーション構築は複数の優位性を提供します：

- **統合アーキテクチャ**：構造化分析、全文検索、ベクトル検索を単一プラットフォームで処理、データ移行と異種システム統合を排除
- **ハイブリッドクエリ性能**：単一SQLでベクトル類似性検索、キーワードフィルタリング、集約分析を同時実行、優れたクエリ性能
- **柔軟スキーマサポート**：VARIANT型が動的JSON構造をネイティブサポート、Light Schema Changeが秒レベルのフィールド・インデックス変更を実現
- **フルスタック最適化**：転置インデックスとベクトルインデックスからMPP実行エンジンまでのエンドツーエンド最適化、検索精度と分析効率のバランス

## レイクハウス for AI

AIモデル・アプリケーション開発では、大量データセットから訓練セットを準備し、特徴量エンジニアリングを実行し、データ品質を評価する必要があります。従来のアーキテクチャでは、データレイクと分析エンジン間で頻繁なデータ移行が必要となることが多くありました。レイクハウスアーキテクチャは、データレイクのオープンストレージとリアルタイム分析エンジンを深く統合し、データ準備、特徴量エンジニアリング、モデル評価の全ワークフローを統合プラットフォーム上でサポートし、データサイロを排除してAI開発イテレーションを加速します。

- **レイクハウス統合アーキテクチャ**：オープンテーブルフォーマット（Iceberg/Paimonなど）とカタログに基づくオープンlakehouseを構築、分析データとAIデータを統一管理
- **リアルタイム分析エンジン**：Dorisがリアルタイム分析エンジンとして機能、インタラクティブクエリと軽量ETLをサポート、データ準備と特徴量エンジニアリングに最速のSQL計算能力を提供
- **シームレスなデータフロー**：データ移動なしでデータレイクを直接読み書き、ストレージレイヤーでの統一管理と計算レイヤーでの柔軟な加速

DorisベースのレイクハウスアーキテクチャはAI全ワークフローを加速します：

- **大規模データ準備**：Dorisの効率的なデータ処理能力を活用し、PBスケールデータレイクからデータをフィルタリング、サンプリング、クレンジングして高品質訓練データセットを迅速構築
- **リアルタイム特徴量エンジニアリング**：Dorisのリアルタイム分析能力を利用してオンライン特徴抽出、変換、集約計算を実行、モデル訓練・推論にリアルタイム特徴サービスを提供
- **品質評価**：テストセットと本番データに対する多次元高速分析を実施、モデル性能とデータドリフトを継続監視

## RAG（Retrieval-Augmented Generation）

RAGは外部知識ベースから関連情報を検索して大規模モデルにコンテキストを提供し、モデルのハルシネーションと知識の通用性問題を効果的に解決します。ベクトルエンジンはRAGシステムのコアコンポーネントであり、大量知識ベースから最も関連性の高いドキュメント断片を迅速に想起しつつ、高並行ユーザークエリリクエストをサポートしてアプリケーションの応答性を保証する必要があります。

- **企業知識**：社内文書とマニュアルに基づくインテリジェントQ&Aシステム構築、従業員が自然言語で迅速に正確な回答を取得
- **インテリジェントカスタマーサービスアシスタント**：製品知識ベースと過去事例を組み合わせ、カスタマーサービス担当者やチャットボットに正確な応答提案を提供
- **インテリジェント文書アシスタント**：大規模文書コレクション内の関連コンテンツを迅速に特定し、研究、執筆、意思決定プロセスを支援

これらのシナリオでDorisベースのRAGアプリケーション構築は以下の優位性を提供します：

- **高並行性能**：分散アーキテクチャが高並行ベクトル検索をサポート、大規模並行ユーザーアクセスを容易に処理
- **ハイブリッド検索能力**：単一SQLでベクトル類似性検索とキーワードフィルタリングを同時実行、セマンティック想起と完全一致のバランス
- **弾性スケーリング**：クエリ性能がクラスター拡張に対して線形スケール、数百万から数百億ベクトルへシームレス移行
- **統合ソリューション**：ベクトルデータ、原文書、ビジネスデータを統一管理、RAGアプリケーションのデータアーキテクチャを簡素化

## AI可観測性

AIモデル訓練イテレーションとアプリケーション運用は大量のログ、メトリクス、トレーシングデータを生成します。問題を正確に特定し性能を継続的に最適化するため、可観測性システムはAIインフラストラクチャの重要コンポーネントとなっています。ビジネス規模拡大に伴い、可観測性プラットフォームはPBスケールデータの高スループット書き込み、ミリ秒レベル検索応答、コスト制御など複数の課題に直面しています。典型的な使用例：

- **モデル訓練監視**：訓練メトリクスとリソース消費をリアルタイム追跡、訓練異常と性能ボトルネックを迅速特定
- **推論サービストレーシング**：各推論リクエストの完全トレースを記録、レイテンシ源とエラーパターンを分析
- **AI**アプリケーションログ分析**：大量アプリケーションログの全文検索と集約分析、トラブルシューティングと行動インサイトをサポート

DorisによるAI可観測性構築は以下の優位性を提供します：

- **究極性能**：PB/day（10GB/s）の持続書き込みをサポート、転置インデックスがログ検索を加速し秒レベル応答
- **コスト最適化**：5:1から10:1の圧縮率、ストレージコスト50%-80%削減、コールドデータの低コストストレージをサポート
- **柔軟スキーマ**：Light Schema Changeが秒レベルフィールド変更を実現、VARIANT型が動的JSON構造をネイティブサポート
- **エコシステムフレンドリー**：OpenTelemetryとELKエコシステムと互換、Grafana/Kibana可視化ツールとの統合をサポート

## セマンティック検索

セマンティック検索はベクトル化技術を通じてテキストの深い意味を捉えます。クエリ用語が文書の表現と異なっていても、セマンティックに関連するコンテンツを検索できます。これは多言語検索、同義語認識、意図理解などのシナリオにおいて重要であり、検索再現率とユーザー体験を大幅に向上させます。典型的な使用例：

- **企業文書検索**：従業員が自然言語で問題を記述し、システムが意図を理解して大量文書から セマンティックに関連するポリシー、手順、知識を想起
- **Eコマース商品検索**：ユーザーが「夏に適した通気性の良い靴」と入力し、システムがニーズを理解して単純なキーワードマッチングではなく関連商品を想起
- **コンテンツ推薦**：記事と動画のセマンティック類似性に基づくインテリジェント推薦、異なる表現で潜在的に興味深いコンテンツを発見

Dorisベースのセマンティック検索アプリケーション構築は以下の優位性を提供します：

- **高性能ベクトル検索**：HNSWとIVFアルゴリズムをサポート、億スケールベクトルで秒以下応答、大規模セマンティック検索要件を容易に処理
- **拡張ハイブリッド検索**：単一SQLでセマンティック検索とキーワードフィルタリングを統合、必要語彙ヒットを確保しつつセマンティック関連コンテンツを想起
- **マルチモーダル拡張**：テキストセマンティック検索のみならず画像・音声等のマルチモーダルコンテンツのセマンティック検索まで拡張可能
- **柔軟量子化最適化**：SQ/PQ量子化技術により、検索精度を維持しつつストレージと計算コストを大幅削減
