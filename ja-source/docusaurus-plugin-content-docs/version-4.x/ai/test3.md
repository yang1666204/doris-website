AI技術が前例のないペースで進歩し続ける中、データインフラストラクチャは現代のAIアプリケーションの基盤となっています。Apache Dorisは、高性能なリアルタイム分析データベースであり、全文検索、ベクトル検索、AI機能、およびMCPベースのインテリジェント相互作用のネイティブ統合を提供します。これらの機能が一体となって、ストレージ、検索、分析にわたる包括的なAIデータスタックを形成します。

- [全文検索](text-search/overview.md)
- [ベクトル検索](vector-search/overview.md)
- [AI機能](ai-function-overview.md)
- [Doris MCP サーバー](https://github.com/apache/doris-mcp-server)

Dorisは、ハイブリッド検索・分析、エージェント向けデータ分析、セマンティック検索、RAGアプリケーション開発、大規模AIシステムの可観測性など、幅広いAI駆動ワークロードに対して、統一された高性能かつコスト効率的なソリューションを提供します。

## エージェント向け分析

AI エージェント技術の普及により、ますます多くの分析的意思決定がAIによって自動的に完了されるようになり、データプラットフォームには究極のリアルタイムパフォーマンスと高い同時実行性能が求められています。従来の「手動分析」とは異なり、エージェント向け分析では、データクエリと意思決定をミリ秒スケールで完了し、大量のエージェントからの同時アクセスをサポートする必要があります。代表的なシナリオには、リアルタイム不正検出、インテリジェント広告配信、パーソナライズドレコメンデーションなどがあります。

Dorisは、高性能MPPアーキテクチャにより、これらのエージェント向け分析シナリオで優れた利点を示します：

- **リアルタイム取り込み・更新**: エージェントの決定が最新データに基づくことを保証、〜1秒の最小データレイテンシ

- **高速分析**: 平均クエリレイテンシ < 100ms、エージェントのリアルタイム意思決定要件を満たす
- **高同時実行クエリ**: 10,000+ QPSをサポート、大量のエージェント同時クエリを容易に処理
- **ネイティブエージェント統合**: MCP サーバーを通じてAI エージェントとシームレスに統合し、開発・統合ワークフローを簡素化

## ハイブリッド検索・分析処理

![img](/images/vector-search/image-5.png)

半構造化データと非構造化データは、データ分析において第一級の市民となっています。顧客レビュー、チャットログ、プロダクションログ、車両信号、その他のデータは、ビジネス意思決定プロセスに深く統合されています。従来の構造化分析ソリューションには、全文検索とベクトル検索機能を組み込み、セマンティック検索をサポートしながら、同じプラットフォーム上で多次元分析と集約統計を実現することが必要です。例えば：

- **顧客インサイト**: レビューテキスト検索とユーザー行動分析を組み合わせて、顧客ニーズと満足度トレンドを正確に特定
- **スマート製造**: プロダクションログの全文検索、機器画像認識、IoTメトリック分析を統合して、故障予測と品質最適化を実現
- **インターネット・オブ・ビークルズ**: 車両信号データ分析、ユーザーフィードバックテキストマイニング、運転行動ベクトル検索を総合して、スマートコクピット体験を向上

Dorisの高性能リアルタイム分析、テキストインデックス、ベクトルインデックス機能に基づいて上記シナリオのAIアプリケーションを構築することで、複数の利点が得られます：

- **統一アーキテクチャ**: 単一プラットフォーム上で構造化分析、全文検索、ベクトル検索を処理し、データ移行と異種システム統合を排除
- **ハイブリッドクエリパフォーマンス**: 単一SQLでベクトル類似検索、キーワードフィルタリング、集約分析を同時に実行し、優秀なクエリパフォーマンスを実現
- **柔軟なスキーマサポート**: VARIANTタイプが動的JSON構造をネイティブサポート、Light Schema Changeが秒レベルのフィールドとインデックス変更を可能にする
- **フルスタック最適化**: 転置インデックスとベクトルインデックスからMPP実行エンジンまでのエンドツーエンド最適化により、検索精度と分析効率のバランスを実現

## AI向けレイクハウス

AIモデルとアプリケーション開発には、大量のデータセットから訓練セットの準備、特徴エンジニアリングの実行、データ品質の評価が必要です。従来のアーキテクチャでは、データレイクと分析エンジン間で頻繁なデータ移行が必要になることが多くあります。レイクハウスアーキテクチャは、データレイクのオープンストレージとリアルタイム分析エンジンを深く統合し、統一プラットフォーム上でデータ準備、特徴エンジニアリング、モデル評価の全ワークフローをサポートし、データサイロを排除してAI開発イテレーションを加速します。

- **レイクハウス統一アーキテクチャ**: オープンテーブルフォーマット（Iceberg/Paimonなど）とカタログに基づいてオープンなレイクハウスを構築し、分析データとAIデータを統一管理
- **リアルタイム分析エンジン**: Dorisがリアルタイム分析エンジンとして機能し、インタラクティブクエリと軽量ETLをサポート、データ準備と特徴エンジニアリングに最高速のSQL計算機能を提供
- **シームレスなデータフロー**: データ移動なしでデータレイクの直接読み書き、ストレージ層での統一管理と計算層での柔軟な高速化

Dorisベースのレイクハウスアーキテクチャは、AI全ワークフローを加速します：

- **大規模データ準備**: Dorisの効率的なデータ処理機能を活用してPBスケールのデータレイクからデータをフィルタリング、サンプリング、クレンジングし、高品質な訓練データセットを迅速に構築
- **リアルタイム特徴エンジニアリング**: Dorisのリアルタイム分析機能を利用してオンライン特徴抽出、変換、集約計算を実行し、モデル訓練と推論にリアルタイム特徴サービスを提供
- **品質評価**: テストセットと本番データに対して多次元高速分析を実行し、モデルパフォーマンスとデータドリフトを継続的に監視

## RAG (Retrieval-Augmented Generation)

RAGは外部ナレッジベースから関連情報を検索して大規模モデルにコンテキストを提供し、モデルの幻覚と知識の時事性問題を効果的に解決します。ベクトルエンジンはRAGシステムのコアコンポーネントであり、大量のナレッジベースから最も関連性の高い文書断片を迅速に呼び出すと同時に、高同時実行ユーザークエリリクエストをサポートしてアプリケーションの応答性を確保する必要があります。

- **企業ナレッジ**: 内部文書とマニュアルに基づいてインテリジェントQ&Aシステムを構築し、従業員が自然言語を通じて正確な回答を迅速に取得することを可能にする
- **インテリジェント顧客サービスアシスタント**: 製品ナレッジベースと履歴ケースを組み合わせて、顧客サービス担当者またはチャットボットに正確な応答提案を提供
- **インテリジェント文書アシスタント**: 大規模文書コレクション内の関連コンテンツを迅速に特定し、研究、執筆、意思決定プロセスを支援

これらのシナリオにおいてDorisベースでRAGアプリケーションを構築することで、以下の利点が得られます：

- **高同時実行パフォーマンス**: 分散アーキテクチャが高同時実行ベクトル検索をサポートし、大規模同時ユーザーアクセスを容易に処理
- **ハイブリッド検索機能**: 単一SQLでベクトル類似検索とキーワードフィルタリングを同時に実行し、セマンティック想起と完全一致のバランスを実現
- **弾性スケーリング**: クエリパフォーマンスがクラスター拡張と線形にスケールし、数百万から数百億のベクトルへシームレスに移行
- **統一ソリューション**: ベクトルデータ、元文書、ビジネスデータを統一管理し、RAGアプリケーションのデータアーキテクチャを簡素化

## AI可観測性

AIモデル訓練イテレーションとアプリケーション運用は、大量のログ、メトリック、トレーシングデータを生成します。問題を正確に特定し、パフォーマンスを継続的に最適化するために、可観測性システムはAIインフラストラクチャの重要なコンポーネントとなっています。ビジネス規模の拡大に伴い、可観測性プラットフォームはPBスケールデータの高スループット書き込み、ミリ秒レベルの検索応答、コスト制御など、複数の課題に直面します。代表的なユースケースには以下があります：

- **モデル訓練監視**: 訓練メトリックとリソース消費のリアルタイム追跡、訓練異常とパフォーマンスボトルネックの迅速な特定
- **推論サービストレーシング**: 各推論リクエストの完全なトレースを記録し、レイテンシソースとエラーパターンを分析
- **AIアプリケーションログ分析**: 大量のアプリケーションログの全文検索と集約分析、トラブルシューティングと行動洞察をサポート

DorisでAI可観測性を構築することで、以下の利点が得られます：

- **究極のパフォーマンス**: PB/日（10GB/s）の持続書き込みをサポート、転置インデックスがログ検索を加速し秒レベルの応答を実現
- **コスト最適化**: 5:1から10:1の圧縮率、ストレージコストを50%-80%削減、コールドデータの低コストストレージをサポート
- **柔軟なスキーマ**: Light Schema Changeが秒レベルのフィールド変更を可能にし、VARIANTタイプが動的JSON構造をネイティブサポート
- **エコシステムフレンドリー**: OpenTelemetryとELKエコシステムと互換性があり、Grafana/Kibana可視化ツールとの統合をサポート

## セマンティック検索

セマンティック検索は、ベクトル化技術を通じてテキストの深い意味を捉えます。クエリ用語が文書の表現と異なっていても、セマンティックに関連するコンテンツを検索することができます。これは、言語横断検索、同義語認識、意図理解などのシナリオにとって重要であり、検索想起率とユーザーエクスペリエンスを大幅に向上させます。代表的なユースケースには以下があります：

- **企業文書検索**: 従業員が自然言語で問題を説明し、システムが意図を理解して大量の文書からセマンティックに関連するポリシー、手順、ナレッジを想起
- **Eコマース製品検索**: ユーザーが「夏に適した通気性の良い靴」と入力し、システムがニーズを理解して単なるキーワードマッチングではなく関連製品を想起
- **コンテンツレコメンデーション**: 記事と動画のセマンティック類似性に基づくインテリジェントレコメンデーション、異なる表現でも興味のある可能性があるコンテンツを発見

Dorisベースでセマンティック検索アプリケーションを構築することで、以下の利点が得られます：

- **高性能ベクトル検索**: HNSWとIVFアルゴリズムをサポート、億規模ベクトルに対して秒未満の応答、大規模セマンティック検索要件を容易に処理
- **強化されたハイブリッド検索**: 単一SQLでセマンティック検索とキーワードフィルタリングを統合し、必要な語彙ヒットを確保しながらセマンティックに関連するコンテンツを想起
- **マルチモーダル拡張**: テキストセマンティック検索のみならず、画像、音声などのマルチモーダルコンテンツのセマンティック検索にも拡張可能
- **柔軟な量子化最適化**: SQ/PQ量子化技術により、検索精度を維持しながらストレージと計算コストを大幅に削減
