AI技術が前例のない速度で進歩し続ける中、データインフラストラクチャは現代のAIアプリケーションの基盤となっています。Apache Dorisは高性能なリアルタイム分析データベースであり、full-text search、vector search、AI functions、MCPベースのインテリジェントな相互作用をネイティブに統合しています。これらの機能は、ストレージ、検索、分析にまたがる包括的なAIデータスタックを形成しています。

- [full-text search](text-search/overview.md)
- [vector search](vector-search/overview.md)
- [AI functions](ai-function-overview.md)
- [Doris MCP Server](https://github.com/apache/doris-mcp-server)

Dorisは、hybrid searchと分析、agent向けデータ分析、セマンティック検索、RAGアプリケーション開発、大規模AIシステムの観測可能性など、幅広いAI駆動ワークロード向けに統一された高性能でコスト効率の良いソリューションを提供します。

## Agent向け分析

AI Agent技術の台頭により、ますます多くの分析的な意思決定がAIによって自動的に完了されるようになり、データプラットフォームには究極のリアルタイム性能と高い同時実行機能の提供が求められています。従来の「手動分析」とは異なり、Agent向け分析では、データクエリと意思決定をミリ秒スケールで完了し、大量のAgentからの同時アクセスをサポートする必要があります。典型的なシナリオには、リアルタイム詐欺検出、インテリジェント広告配置、パーソナライズされた推奨などがあります。

Dorisは、高性能なMPPアーキテクチャにより、これらのagent向け分析シナリオで卓越した優位性を示しています：

- **リアルタイム取り込みと更新**: Agentの意思決定が最新のデータに基づくことを保証し、最小データ遅延は約1秒

- **超高速分析**: 平均クエリ遅延 < 100ms、Agentのリアルタイム意思決定要件を満たします
- **高同時実行クエリ**: 10,000+ QPSをサポートし、大量のAgent同時クエリを容易に処理
- **ネイティブAgent統合**: MCP Serverを通じてAI Agentsとシームレスに統合し、開発と統合のワークフローを簡素化

## Hybrid SearchとAnalytics Processing

![img](/images/vector-search/image-5.png)

半構造化データと非構造化データは、データ分析において第一級市民になりつつあります。顧客レビュー、チャットログ、本番ログ、車両信号、その他のデータが、ビジネス意思決定プロセスに深く統合されています。従来の構造化分析ソリューションは、full-text検索とvector search機能を組み込み、セマンティック検索をサポートしながら、同じプラットフォーム上で多次元分析と集計統計を可能にする必要があります。例として以下があります：

- **顧客インサイト**: レビューテキスト検索とユーザー行動分析を組み合わせて、顧客ニーズと満足度トレンドを正確に特定
- **スマート製造**: 本番ログのfull-text search、設備画像認識、IoTメトリクス分析を統合して、故障予測と品質最適化を実現
- **Internet of Vehicles**: 車両信号データ分析、ユーザーフィードバックテキストマイニング、運転行動vector検索を統合して、スマートコックピット体験を向上

Dorisの高性能リアルタイム分析、テキストインデックス、vectorインデックス機能に基づいて上記シナリオ向けAIアプリケーションを構築することで、複数の利点が得られます：

- **統一アーキテクチャ**: 単一プラットフォーム上で構造化分析、full-text検索、vector searchを処理し、データ移行と異種システム統合を排除
- **Hybridクエリ性能**: 単一SQLでvector類似検索、キーワードフィルタリング、集計分析を同時に実行し、優れたクエリ性能を実現
- **柔軟なスキーマサポート**: VARIANT型が動的JSON構造をネイティブサポート、Light Schema Changeが秒レベルのフィールドとインデックス変更を可能に
- **フルスタック最適化**: inverted indexesとvector indexesからMPP実行エンジンまでのエンドツーエンド最適化で、検索精度と分析効率のバランスを実現

## AI向けLakehouse

AIモデルとアプリケーション開発には、大量のデータセットから訓練セットの準備、特徴エンジニアリングの実行、データ品質の評価が必要です。従来のアーキテクチャでは、データレイクと分析エンジン間での頻繁なデータ移行が必要でした。Lakehouseアーキテクチャは、データレイクのオープンストレージとリアルタイム分析エンジンを深く統合し、データ準備、特徴エンジニアリング、モデル評価の全ワークフローを統一プラットフォーム上でサポートし、データサイロを排除してAI開発の反復を加速します。

- **Lakehouse統一アーキテクチャ**: オープンテーブル形式（IcebergやPaimonなど）とCatalogsに基づいてオープンlakehouseを構築し、分析データとAIデータを統一管理
- **リアルタイム分析エンジン**: Dorisがリアルタイム分析エンジンとして機能し、対話型クエリと軽量ETLをサポートし、データ準備と特徴エンジニアリング向けに最速のSQL計算機能を提供
- **シームレスなデータフロー**: データ移動なしでデータレイクを直接読み書きし、ストレージレイヤーでの統一管理と計算レイヤーでの柔軟な加速を実現

Dorisベースの Lakehouseアーキテクチャは、AI全体のワークフローを加速します：

- **大規模データ準備**: Dorisの効率的なデータ処理機能を活用してPBスケールのデータレイクからデータをフィルタリング、サンプリング、クレンジングし、高品質な訓練データセットを迅速に構築
- **リアルタイム特徴エンジニアリング**: Dorisのリアルタイム分析機能を活用してオンライン特徴抽出、変換、集計計算を実行し、モデル訓練と推論向けにリアルタイム特徴サービスを提供
- **品質評価**: テストセットと本番データに対する多次元迅速分析を実施し、モデル性能とデータドリフトを継続的に監視

## RAG (Retrieval-Augmented Generation)

RAGは外部知識ベースから関連情報を検索して大型モデルにコンテキストを提供し、モデルの幻覚と知識の時事性の問題を効果的に解決します。vectorエンジンはRAGシステムの中核コンポーネントであり、大量の知識ベースから最も関連性の高い文書フラグメントを迅速に呼び出し、同時に高同時実行ユーザークエリリクエストをサポートしてアプリケーションの応答性を確保する必要があります。

- **企業知識**: 内部文書とマニュアルに基づいてインテリジェントQ&Aシステムを構築し、従業員が自然言語を通じて正確な回答を迅速に取得できるように支援
- **インテリジェント顧客サービスアシスタント**: 製品知識ベースと過去の事例を組み合わせて、カスタマーサービス担当者やチャットボット向けに正確な応答提案を提供
- **インテリジェント文書アシスタント**: 大規模文書コレクション内の関連コンテンツを迅速に特定し、研究、執筆、意思決定プロセスを支援

これらのシナリオでDorisに基づいてRAGアプリケーションを構築することで、以下の利点が得られます：

- **高同時実行性能**: 分散アーキテクチャが高同時実行vector検索をサポートし、大規模同時ユーザーアクセスを容易に処理
- **Hybrid検索機能**: 単一SQLでvector類似検索とキーワードフィルタリングを同時に実行し、セマンティック再現と完全一致のバランスを実現
- **弾性スケーリング**: クエリ性能がクラスター拡張と共に線形にスケールし、数百万から数百億のvectorへシームレスに移行
- **統一ソリューション**: vectorデータ、元文書、ビジネスデータを統一管理し、RAGアプリケーションのデータアーキテクチャを簡素化

## AI観測可能性

AIモデル訓練の反復とアプリケーション運用は、大量のログ、メトリクス、トレースデータを生成します。問題を正確に特定し、性能を継続的に最適化するために、観測可能性システムはAIインフラストラクチャの重要なコンポーネントとなっています。ビジネススケールの拡大に伴い、観測可能性プラットフォームは、PBスケールデータの高スループット書き込み、ミリ秒レベルの検索応答、コスト制御など、複数の課題に直面しています。典型的なユースケースには以下があります：

- **モデル訓練監視**: 訓練メトリクスとリソース消費のリアルタイム追跡、訓練異常と性能ボトルネックの迅速な特定
- **推論サービストレース**: 各推論リクエストの完全なトレースを記録し、遅延源とエラーパターンを分析
- **AI** **アプリケーションログ分析**: 大量のアプリケーションログのfull-text検索と集計分析、トラブルシューティングと行動インサイトをサポート

DorisでAI観測可能性を構築することで、以下の利点が得られます：

- **究極の性能**: PB/日（10GB/s）の持続書き込みをサポート、inverted indexesがログ検索を加速して秒レベルの応答を実現
- **コスト最適化**: 5:1から10:1の圧縮率、ストレージコスト50%-80%削減、コールドデータ向け低コストストレージをサポート
- **柔軟なスキーマ**: Light Schema Changeが秒レベルのフィールド変更を可能に、VARIANT型が動的JSON構造をネイティブサポート
- **エコシステムフレンドリー**: OpenTelemetryとELKエコシステムと互換性があり、Grafana/Kibana可視化ツールとの統合をサポート

## セマンティック検索

セマンティック検索は、ベクトル化技術を通じてテキストの深い意味を捉えます。クエリ用語が文書の文言と異なっていても、意味的に関連性のあるコンテンツを検索できます。これは、言語横断検索、同義語認識、意図理解などのシナリオにおいて重要であり、検索再現率とユーザー体験を大幅に改善します。典型的なユースケースには以下があります：

- **企業文書検索**: 従業員が自然言語で問題を記述し、システムが意図を理解して大量の文書から意味的に関連性のあるポリシー、手順、知識を再現
- **eコマース製品検索**: ユーザーが「夏に適した通気性の良い靴」と入力すると、システムがニーズを理解して単なるキーワードマッチングではなく関連製品を再現
- **コンテンツ推奨**: 記事と動画のセマンティック類似性に基づくインテリジェント推奨で、異なる文言でも潜在的に興味のあるコンテンツを発見

Dorisに基づいてセマンティック検索アプリケーションを構築することで、以下の利点が得られます：

- **高性能vector検索**: HNSWとIVFアルゴリズムをサポート、億スケールのvectorに対して秒未満の応答、大規模セマンティック検索要件を容易に処理
- **強化されたhybrid検索**: 単一SQLでセマンティック検索とキーワードフィルタリングを統合し、必要な語彙ヒットを確保しながら意味的に関連性のあるコンテンツを再現
- **マルチモーダル拡張**: テキストセマンティック検索のみならず、画像や音声などのマルチモーダルコンテンツのセマンティック検索にも拡張可能
- **柔軟な量子化最適化**: SQ/PQ量子化技術により、検索精度を維持しながらストレージと計算コストを大幅に削減
