AI技術が前例のないスピードで進歩し続ける中、データインフラストラクチャは現代のAIアプリケーションの礎となっています。高性能リアルタイム分析データベースであるApache Dorisは、全文検索、ベクター検索、AI機能、およびMCPベースのインテリジェント対話のネイティブ統合を提供します。これらの機能は一体となって、ストレージ、検索、および分析を包括する総合的なAIデータスタックを形成します。

- [full-text search](text-search/overview.md)
- [vector search](vector-search/overview.md)
- [AI functions](ai-function-overview.md)
- [Doris MCP サーバー](https://github.com/apache/doris-mcp-server)

Dorisは、ハイブリッド検索と分析、エージェント向けデータ分析、セマンティック検索、RAGアプリケーション開発、および大規模AIシステムの可観測性など、幅広いAI駆動ワークロードに対して統一された高性能でコスト効率的なソリューションを提供します。

## エージェント Facing Analytics

AI エージェント技術の台頭により、ますます多くの分析的意思決定がAIによって自動的に完了されるようになり、データプラットフォームには究極のリアルタイムパフォーマンスと高い同時実行機能の提供が求められています。従来の「手動分析」とは異なり、エージェント Facing Analyticsでは、データクエリと意思決定をミリ秒スケールで完了し、大量のエージェントからの同時アクセスをサポートする必要があります。典型的なシナリオには、リアルタイム不正検知、インテリジェント広告配置、パーソナライズドレコメンデーションが含まれます。

Dorisは、高性能MPPアーキテクチャにより、これらのエージェント向け分析シナリオで優れた優位性を実証しています：

- **リアルタイム取り込み＆更新**：エージェントの意思決定が最新データに基づくことを保証、最小データレイテンシ～1秒

- **超高速分析**：平均クエリレイテンシ < 100ms、エージェントのリアルタイム意思決定要件を満たす
- **高同時実行クエリ**：10,000+ QPSをサポート、大量のエージェント同時クエリを容易に処理
- **ネイティブエージェント統合**：MCP サーバーを通じてAI エージェントとシームレスに統合、開発と統合ワークフローを簡素化

## ハイブリッド検索と分析処理

![img](/images/vector-search/image-5.png)

半構造化および非構造化データは、データ分析における第一級市民になりつつあります。顧客レビュー、チャットログ、本番ログ、車両信号、その他のデータは、ビジネス意思決定プロセスに深く統合されています。従来の構造化分析ソリューションでは、全文検索とベクター検索機能を組み込み、セマンティック検索をサポートしながら、同一プラットフォーム上で多次元分析と集約統計を可能にする必要があります。例には以下が含まれます：

- **顧客インサイト**：レビューテキスト検索とユーザー行動分析を組み合わせ、顧客ニーズと満足度トレンドを精密に特定
- **スマート製造**：本番ログ全文検索、設備画像認識、IoTメトリクス分析を統合し、故障予測と品質最適化を実現
- **コネクテッドカー**：車両信号データ分析、ユーザーフィードバックテキストマイニング、運転行動ベクター検索を統合し、スマートコックピット体験を向上

Dorisの高性能リアルタイム分析、テキストインデックス、ベクターインデックス機能に基づいて上記シナリオ向けAIアプリケーションを構築することで、複数の利点を提供します：

- **統一アーキテクチャ**：単一プラットフォームで構造化分析、全文検索、ベクター検索を処理、データ移行と異種システム統合を排除
- **ハイブリッドクエリパフォーマンス**：単一SQLでベクター類似検索、キーワードフィルタリング、集約分析を同時実行し、優秀なクエリパフォーマンスを実現
- **柔軟なスキーマサポート**：VARIANT型は動的JSON構造をネイティブサポート、Light Schema Changeは秒レベルのフィールドとインデックス変更を可能に
- **フルスタック最適化**：転置インデックスとベクターインデックスからMPP実行エンジンまでのエンドツーエンド最適化、検索精度と分析効率のバランスを実現

## レイクハウス for AI

AIモデルとアプリケーション開発では、大量データセットからトレーニングセットの準備、特徴エンジニアリングの実行、データ品質の評価が必要です。従来のアーキテクチャでは、データレイクと分析エンジン間で頻繁なデータ移行が必要でした。レイクハウスアーキテクチャは、データレイクのオープンストレージとリアルタイム分析エンジンを深く統合し、データ準備、特徴エンジニアリング、モデル評価の全ワークフローを統一プラットフォーム上でサポートし、データサイロを排除してAI開発イテレーションを加速します。

- **レイクハウス統一アーキテクチャ**：オープンテーブルフォーマット（Iceberg/Paimonなど）とカタログに基づくオープンlakehouseを構築、分析データとAIデータを統一管理
- **リアルタイム分析エンジン**：Dorisがリアルタイム分析エンジンとして機能し、対話型クエリと軽量ETLをサポート、データ準備と特徴エンジニアリングに最高速のSQL計算能力を提供
- **シームレスなデータフロー**：データ移動なしでデータレイクへの直接読み書き、ストレージレイヤでの統一管理とコンピュートレイヤでの柔軟な高速化

Dorisに基づくレイクハウスアーキテクチャは、AI全体のワークフローを加速します：

- **大規模データ準備**：DorisのデータプレEfficientでPBスケールのデータレイクからデータをフィルタリング、サンプリング、クレンジングし、高品質トレーニングデータセットを迅速に構築
- **リアルタイム特徴エンジニアリング**：Dorisのリアルタイム分析能力を活用してオンライン特徴抽出、変換、集約計算を実行し、モデルトレーニングと推論にリアルタイム特徴サービスを提供
- **品質評価**：テストセットと本番データに対して多次元高速分析を実施し、モデルパフォーマンスとデータドリフトを継続的に監視

## RAG (Retrieval-Augmented Generation)

RAGは外部知識ベースから関連情報を取得して大規模モデルにコンテキストを提供し、モデルの幻覚と知識の時効性問題を効果的に解決します。ベクターエンジンはRAGシステムのコアコンポーネントであり、大量知識ベースから最も関連性の高い文書断片を迅速に呼び出し、同時に高同時実行ユーザークエリリクエストをサポートしてアプリケーションの応答性を保証する必要があります。

- **企業知識**：内部文書とマニュアルに基づくインテリジェントQ&Aシステムの構築、従業員が自然言語を通じて正確な回答を迅速に取得可能
- **インテリジェントカスタマーサービスアシスタント**：製品知識ベースと履歴事例を組み合わせ、カスタマーサービス担当者やチャットボットに精密な応答提案を提供
- **インテリジェント文書アシスタント**：大規模文書コレクションで関連コンテンツを迅速に特定し、研究、執筆、意思決定プロセスを支援

これらのシナリオでDorisに基づいてRAGアプリケーションを構築することで、以下の利点を提供します：

- **高同時実行パフォーマンス**：分散アーキテクチャが高同時実行ベクター検索をサポート、大規模同時ユーザーアクセスを容易に処理
- **ハイブリッド検索機能**：単一SQLでベクター類似検索とキーワードフィルタリングを同時実行、セマンティック呼び出しと完全一致のバランスを実現
- **弾性スケーリング**：クラスタ拡張に伴いクエリパフォーマンスが線形スケール、数百万から数百億ベクターへのシームレスな移行
- **統一ソリューション**：ベクターデータ、元文書、ビジネスデータを統一管理し、RAGアプリケーションのデータアーキテクチャを簡素化

## AI可観測性

AIモデルトレーニングイテレーションとアプリケーション運用は大量のログ、メトリクス、トレーシングデータを生成します。問題を精密に特定し、パフォーマンスを継続的に最適化するため、可観測性システムはAIインフラストラクチャの重要なコンポーネントになっています。ビジネス規模の拡大に伴い、可観測性プラットフォームはPBスケールデータの高スループット書き込み、ミリ秒レベル検索レスポンス、コスト制御など複数の課題に直面します。典型的なユースケースには以下が含まれます：

- **モデルトレーニング監視**：トレーニングメトリクスとリソース消費のリアルタイム追跡、トレーニング異常とパフォーマンスボトルネックの迅速特定
- **推論サービストレーシング**：各推論リクエストの完全トレースを記録、レイテンシソースとエラーパターンを分析
- **AI****アプリケーションログ分析**：大量アプリケーションログの全文検索と集約分析、トラブルシューティングと行動インサイトをサポート

DorisでAI可観測性を構築することで、以下の利点を提供します：

- **究極のパフォーマンス**：PB/日（10GB/s）の持続書き込みをサポート、転置インデックスがログ検索を加速し秒レベルレスポンスを実現
- **コスト最適化**：5:1から10:1の圧縮率、ストレージコスト50%-80%削減、コールドデータ用低コストストレージをサポート
- **柔軟なスキーマ**：Light Schema Changeが秒レベルフィールド変更を可能に、VARIANT型は動的JSON構造をネイティブサポート
- **エコシステムフレンドリー**：OpenTelemetryとELKエコシステムと互換性があり、Grafana/Kibana可視化ツールとの統合をサポート

## セマンティック検索

セマンティック検索は、ベクター化技術を通じてテキストの深い意味を捉えます。クエリ用語が文書の文言と異なる場合でも、セマンティックに関連するコンテンツを検索できます。これは、多言語検索、同義語認識、意図理解などのシナリオで重要であり、検索再現率とユーザーエクスペリエンスを大幅に向上させます。典型的なユースケースには以下が含まれます：

- **企業文書検索**：従業員が自然言語で問題を記述し、システムが意図を理解して大量文書からセマンティックに関連するポリシー、手順、知識を呼び出し
- **Eコマース商品検索**：ユーザーが「夏に適した通気性のある靴」と入力し、システムがニーズを理解して単なるキーワードマッチングではなく関連商品を呼び出し
- **コンテンツレコメンデーション**：記事と動画のセマンティック類似性に基づくインテリジェントレコメンデーション、異なる文言でも潜在的に興味のあるコンテンツを発見

Dorisに基づいてセマンティック検索アプリケーションを構築することで、以下の利点を提供します：

- **高性能ベクター検索**：HNSWとIVFアルゴリズムをサポート、億スケールベクターで秒未満レスポンス、大規模セマンティック検索要件を容易に処理
- **強化ハイブリッド検索**：単一SQLでセマンティック検索とキーワードフィルタリングを統合、必要な語彙ヒットを保証しながらセマンティックに関連するコンテンツを呼び出し
- **マルチモーダル拡張**：テキストセマンティック検索だけでなく、画像や音声などマルチモーダルコンテンツのセマンティック検索への拡張も可能
- **柔軟な量子化最適化**：SQ/PQ量子化技術を通じて、検索精度を維持しながらストレージと計算コストを大幅に削減
