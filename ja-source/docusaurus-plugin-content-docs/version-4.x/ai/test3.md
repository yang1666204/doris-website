AI技術が前例のないペースで発展し続ける中、データインフラストラクチャは現代のAIアプリケーションの基盤となっています。高性能リアルタイム分析データベースのApache Dorisは、全文検索、ベクトル検索、AI機能、MCPベースのインテリジェントなやり取りのネイティブ統合を提供します。これらの機能が一体となって、ストレージ、検索、分析にわたる包括的なAIデータスタックを形成しています。

- [full-text search](text-search/overview.md)
- [vector search](vector-search/overview.md)
- [AI functions](ai-function-overview.md)
- [Doris MCP サーバー](https://github.com/apache/doris-mcp-server)

Dorisは、ハイブリッド検索と分析、エージェント向けデータ分析、セマンティック検索、RAGアプリケーション開発、大規模AIシステムのオブザーバビリティなど、幅広いAI駆動ワークロードに対して統一された高性能で費用対効果の高いソリューションを提供します。

## エージェント向け分析

AI エージェントテクノロジーの台頭により、分析的な意思決定の多くはAIによって自動的に完了されるようになり、データプラットフォームには究極のリアルタイム性能と高い同時実行能力の提供が求められています。従来の「手動分析」とは異なり、エージェント向け分析では、データクエリと意思決定をミリ秒スケールで完了し、膨大な数のエージェントからの同時アクセスをサポートすることが求められます。典型的なシナリオには、リアルタイム不正検出、インテリジェント広告配信、パーソナライズされたレコメンデーションが含まれます。

Dorisは、高性能MPPアーキテクチャにより、これらのエージェント向け分析シナリオで優れた利点を示します：

- **リアルタイム取り込み・更新**: エージェントの意思決定が最新データに基づくことを保証、最小データレイテンシ約1秒

- **超高速分析**: 平均クエリレイテンシ100ms未満、エージェントのリアルタイム意思決定要件を満たします
- **高同時クエリ**: 10,000+ QPSをサポート、大規模エージェント同時クエリを容易に処理
- **ネイティブエージェント統合**: MCP サーバーを通じてAI エージェントとシームレスに統合、開発と統合ワークフローを簡素化

## ハイブリッド検索と分析処理

![img](/images/vector-search/image-5.png)

半構造化データと非構造化データは、データ分析においてファーストクラスの存在となっています。カスタマーレビュー、チャットログ、本番ログ、車両信号などのデータが、ビジネス意思決定プロセスに深く統合されています。従来の構造化分析ソリューションには、全文検索とベクトル検索機能の組み込みが必要で、セマンティック検索をサポートしながら、同一プラットフォーム上で多次元分析と集約統計を可能にします。例えば以下のようなものがあります：

- **顧客インサイト**: レビューテキスト検索とユーザー行動分析を組み合わせて、顧客ニーズと満足度トレンドを正確に識別
- **スマート製造**: 生産ログの全文検索、設備画像認識、IoTメトリック分析を統合して、故障予測と品質最適化を実現
- **車両IoT**: 車両信号データ分析、ユーザーフィードバックテキストマイニング、運転行動ベクトル検索を統合して、スマートコックピット体験を向上

Dorisの高性能リアルタイム分析、テキストインデックス、ベクトルインデックス機能に基づいて上記シナリオのAIアプリケーションを構築することで、複数の利点が得られます：

- **統一アーキテクチャ**: 構造化分析、全文検索、ベクトル検索を単一プラットフォームで処理、データ移行と異種システム統合を排除
- **ハイブリッドクエリ性能**: 単一SQLでベクトル類似検索、キーワードフィルタリング、集約分析を同時実行し、優れたクエリ性能を実現
- **柔軟なスキーマサポート**: VARIANT型は動的JSON構造をネイティブサポート、Light Schema Changeは秒レベルのフィールドとインデックス変更を可能にします
- **フルスタック最適化**: 転置インデックスとベクトルインデックスからMPP実行エンジンまでのエンドツーエンド最適化、検索精度と分析効率のバランスを実現

## AI向けレイクハウス

AIモデルとアプリケーション開発には、膨大なデータセットからトレーニングセットの準備、特徴エンジニアリングの実行、データ品質の評価が必要です。従来のアーキテクチャでは、データレイクと分析エンジン間の頻繁なデータ移行が必要になることがよくあります。レイクハウスアーキテクチャは、データレイクのオープンストレージとリアルタイム分析エンジンを深く統合し、統一プラットフォーム上でデータ準備、特徴エンジニアリング、モデル評価の全ワークフローをサポートし、データサイロを排除してAI開発イテレーションを加速します。

- **レイクハウス統一アーキテクチャ**: オープンテーブル形式（IcebergやPaimonなど）とカタログに基づいてオープンなlakehouseを構築し、分析データとAIデータを統一管理
- **リアルタイム分析エンジン**: Dorisはリアルタイム分析エンジンとして機能し、インタラクティブクエリと軽量ETLをサポートし、データ準備と特徴エンジニアリングに最高速のSQL計算能力を提供
- **シームレスなデータフロー**: データ移動なしでデータレイクへの直接読み書き、ストレージ層での統一管理とコンピュート層での柔軟な加速

Dorisベースのレイクハウスアーキテクチャは、AI全体のワークフローを加速します：

- **大規模データ準備**: Dorisの効率的なデータ処理能力を活用して、PBスケールのデータレイクからデータをフィルタリング、サンプリング、クレンジングし、高品質なトレーニングデータセットを迅速構築
- **リアルタイム特徴エンジニアリング**: Dorisのリアルタイム分析能力を活用してオンライン特徴抽出、変換、集約計算を実行し、モデル訓練と推論にリアルタイム特徴サービスを提供
- **品質評価**: テストセットと本番データに対して多次元の迅速分析を実行し、モデル性能とデータドリフトを継続的に監視

## RAG（検索拡張生成）

RAGは外部ナレッジベースから関連情報を検索して大規模モデルにコンテキストを提供し、モデルの幻覚と知識の鮮度問題を効果的に解決します。ベクトルエンジンはRAGシステムのコアコンポーネントであり、膨大なナレッジベースから最も関連性の高いドキュメントフラグメントを迅速に呼び出し、同時に高同時実行ユーザークエリ要求をサポートしてアプリケーションの応答性を確保する必要があります。

- **企業ナレッジ**: 内部ドキュメントとマニュアルに基づいたインテリジェントQ&Aシステムを構築し、従業員が自然言語を通じて正確な回答を迅速に取得
- **インテリジェント カスタマーサービスアシスタント**: 製品ナレッジベースと履歴ケースを組み合わせて、カスタマーサービス担当者やチャットボットに正確な対応提案を提供
- **インテリジェント ドキュメントアシスタント**: 大規模ドキュメントコレクションから関連コンテンツを迅速に特定し、研究、執筆、意思決定プロセスを支援

これらのシナリオでDorisベースのRAGアプリケーションを構築することで、以下の利点があります：

- **高同時実行性能**: 分散アーキテクチャが高同時実行ベクトル検索をサポートし、大規模同時ユーザーアクセスを容易に処理
- **ハイブリッド検索能力**: 単一SQLでベクトル類似検索とキーワードフィルタリングを同時実行し、セマンティック呼び出しと完全一致のバランスを実現
- **弾性スケーリング**: クラスター拡張に伴ってクエリ性能が線形スケール、数百万から数百億ベクトルへのシームレスな移行
- **統一ソリューション**: ベクトルデータ、元ドキュメント、ビジネスデータを統一管理し、RAGアプリケーションのデータアーキテクチャを簡素化

## AIオブザーバビリティ

AIモデル訓練イテレーションとアプリケーション運用は、膨大な量のログ、メトリック、トレーシングデータを生成します。問題を正確に特定し性能を継続的に最適化するため、オブザーバビリティシステムはAIインフラストラクチャの重要なコンポーネントとなっています。ビジネス規模の拡大に伴い、オブザーバビリティプラットフォームは、PBスケールデータの高スループット書き込み、ミリ秒レベル検索応答、コスト管理など複数の課題に直面します。典型的なユースケースには以下があります：

- **モデル訓練監視**: 訓練メトリックとリソース消費をリアルタイム追跡し、訓練異常と性能ボトルネックを迅速に特定
- **推論サービストレーシング**: 各推論リクエストの完全なトレースを記録し、レイテンシソースとエラーパターンを分析
- **AI**  **アプリケーションログ分析**: 膨大なアプリケーションログの全文検索と集約分析、トラブルシューティングと行動インサイトをサポート

DorisでAI Observabilityを構築することで、以下の利点があります：

- **究極の性能**: PB/日（10GB/s）の持続書き込みをサポート、転置インデックスがログ検索を加速し秒レベル応答を実現
- **コスト最適化**: 5:1から10:1の圧縮率、50%-80%のストレージコスト削減、コールドデータの低コストストレージをサポート
- **柔軟なスキーマ**: Light Schema Changeが秒レベルフィールド変更を可能にし、VARIANT型は動的JSON構造をネイティブサポート
- **エコシステム対応**: OpenTelemetryとELKエコシステムと互換性があり、Grafana/Kibana可視化ツールとの統合をサポート

## セマンティック検索

セマンティック検索は、ベクトル化技術によりテキストの深い意味を捉えます。クエリ用語がドキュメントの文言と異なっていても、セマンティック的に関連するコンテンツを検索できます。これは、言語横断検索、類義語認識、意図理解などのシナリオで重要であり、検索の再現率とユーザー体験を大幅に向上させます。典型的なユースケースには以下があります：

- **企業ドキュメント検索**: 従業員が自然言語で問題を記述し、システムが意図を理解して膨大なドキュメントからセマンティック的に関連するポリシー、手順、ナレッジを呼び出し
- **ECサイト商品検索**: ユーザーが「夏に適した通気性の良い靴」と入力し、システムがニーズを理解して単純なキーワードマッチングではなく関連商品を呼び出し
- **コンテンツレコメンデーション**: 記事や動画のセマンティック類似性に基づいたインテリジェントレコメンデーション、文言が異なっても潜在的に興味のあるコンテンツを発見

Dorisベースのセマンティック検索アプリケーション構築により、以下の利点があります：

- **高性能ベクトル検索**: HNSWとIVFアルゴリズムをサポート、億規模ベクトルの秒未満応答、大規模セマンティック検索要件を容易に処理
- **強化されたハイブリッド検索**: 単一SQLでセマンティック検索とキーワードフィルタリングを統合し、必要な語彙ヒットを確保しながらセマンティック的に関連するコンテンツを呼び出し
- **マルチモーダル拡張**: テキストセマンティック検索のサポートに加え、画像、音声などのマルチモーダルコンテンツのセマンティック検索にも拡張可能
- **柔軟な量子化最適化**: SQ/PQ量子化技術により、検索精度を維持しながらストレージと計算コストを大幅削減
